## Code of Conduct

By participating in this project, you agree to abide by the Code of Conduct. We expect all contributors to maintain a professional, respectful environment, and strictly respect the original authorship, architecture, and branding of the 𝕏TV project.

## How to Contribute

We welcome contributions in several forms:
-   **Bug Reports**: Please open a detailed issue if you encounter any problems, including logs and reproduction steps.
-   **Feature Requests**: Have a groundbreaking idea (like our Ephemeral Tunnels)? Open an issue to discuss it before starting work!
-   **Pull Requests**: Submit your code improvements via PRs.

### Guidelines for Pull Requests

1.  **Fork the Repository**: Create your own copy of the project.
2.  **Branch Out**: Create a new branch for your feature or bugfix (e.g., `feature/new-audio-editor` or `fix/db-connection`).
3.  **Code Style**: Ensure your code follows the existing style and architecture. If adding new database models or configurations, make sure they support both `PUBLIC_MODE=True` and `PUBLIC_MODE=False`.
4.  **Respect Authorship**: Do not remove or alter the original credits, branding, or copyright notices (e.g., "Developed by 𝕏0L0™", "𝕏TV Pro™").
5.  **Submit a Pull Request**: Provide a clear description of your changes. Link any relevant issues your PR resolves.
6.  **English Only**: Please ensure all discussions, PR descriptions, and code comments are entirely in English to maintain consistency.
